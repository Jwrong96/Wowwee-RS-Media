RSM Alarm Clock V2 config.ini settings
Config file [section] and key Range Default *1 Legal/illegal combinations
[alarmx] x can be 1 - 20.  N/A  
enabled = [Y][[N][1][0] Y/1 = alarm enabled
N/0 = Alarm disabled		 1 (alarm enabled)  
date = YYYY/MM/DD Must be YYYY/MM/DD format.
Year must be > 1900		 empty A time value must be specifed if a date value is specified.
time = HH:MM[am][pm] 12 hour or 24 hour time for alarm to be triggered.
Am or pm and be specified if using 12 hour format.		 empty times can be set in 12hr (using am and pm) or in 24 hour using 00-23 for HH and not specifying am or pm.
recur = xx{s,m,h} xx=an integer. 0 will disable recursion
xx>0 enables the alarm to be retriggered after xx seconds,minutes or hours.		 0 Recur value can be set with these combinations
-date & time  - will trigger on the date/time specified
-time value -  will trigger daily at the specified time.
-no time or date value - will triggerr xx after the program is started.
Recur value must be less than 24hours (it will be ignored if its bigger)
Cannot be used for a weekday alarm (weekday setting will be ignored , alarm will be treated as a daily alarm.) (This is a bug in the V2 program)
weekday = <integer> 0 = disabled
1=Monday,2=Tuesday…….7=Sunday		 0 A time value must be specifed if a weekday value is specified.
Do not specify a recur value with a weekday alarm. (This is a bug in the V2 version)
recurcount = <integer> 0  - unlimited recursion , alarm will trigger every recur period until the progam is stopped. (If it’s a date alarm it will probably stop when the next day occurs)
1 or greater - alarm will be triggered this many times.		 0 Valid with any recurring alarm
action = XXXXX:"<string>" XXXXX matches a action in the [actioncmds] section
<string> represents a valid unix path to a unix directory in RSMedia or SD card. Quotes MUST be used if any spaces or special chars are used in the path.		 empy Only one of action/action_rnd/action_seq should be specified
action_rnd = XXXXX:"<path>" Same as action empy Only one of action/action_rnd/action_seq should be specified
action_seq = XXXXX:"<path>" Same as action empy Only one of action/action_rnd/action_seq should be specified
       
snooze = <integer> Supported in Ini file but NOT used by program 0  
duration = <integer> Supported in Ini file but NOT used by program 0  
volume = <integer> Supported in Ini file but NOT used by program 0  
       
[debug]      
level = <integer> 0=no logging output, 1= error logging output,
2= useful info output, 3= all debugging info output 		 0  
[misc]      
24hourmode = [Y][[N][1][0] Y/1 = alarm enabled
N/0 = Alarm disabled		 N  
yoffset = <integer> 0-95.  0= top , 30 = middle , 95 = bottom 0 Can be used to move the clock digits down the screen
xoffset = <integer> 0-10 0 Can be used to offset the clock digits across the screen
background = <path> path to a jpg/png/gif file that will be used as a background picture for the clock. empty If no background is specified then a black backgroud is used (I think!)
timeadjust = <signed integer> Supported in Ini file but feature in program does bot work. 0 You should not set the value for RSMalarm clock V2.
forceLCDon = [Y][[N][1][0] Y/1 = LCD will be always forced on (ie low power mode after 2 mins will be disabled)
N/0 = LCD will be NOT be forced on (ie lowerpower  mode after 2 mins will still occur)		 1 If set to N then whan an alarm is triggered the LCD should be turned back ON. BUT the remote may not return out of sleep mode , so movement maybe disabled (but sound should still work).
*1 the default is used if the key & value are NOT specified in the ini file.
